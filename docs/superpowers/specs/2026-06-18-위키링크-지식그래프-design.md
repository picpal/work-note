# 위키링크·지식 그래프 (② 코어 연결) 설계

> 작성일 2026-06-18 · 상태: 설계 확정 · 다음 단계: 구현 계획(writing-plans)

## 배경과 목표

work-note는 사내 마크다운 지식저장소다. 향후 방향은 **LLM-WIKI의 자료 근간 + 사내 에이전트 RAG 소스**로 진화하는 것이며, 이를 3개 레이어로 분해했다(권장 순서 ①→②→③):

- **① 신뢰·신선도** — 문서 상태(초안/검증/폐기)·소유자·최종검토일·stale 감지.
- **② 지식 그래프** — `[[위키링크]]`·백링크·템플릿. **(본 스펙)**
- **③ 권한-aware 인제스션 계약** — export API·청크+메타·증분 동기 웹훅·citation (LLM/임베딩 필요).

현재 work-note는 "폴더 트리 + 서로 모르는 문서 섬들"이다. 노트 간 연결 수단이 없고(검색 점프뿐), 지식 그래프의 핵심인 *연결*이 통째로 비어 있다. 본 스펙은 **② 중 코어 연결**만 다룬다.

### 범위 (v1)

**포함:** `[[위키링크]]` 입력·자동완성, 링크 렌더(에디터·미리보기·공유), 백링크 패널, 접근불가/깨진 링크 처리, 백링크 있는 노트 삭제 경고.

**제외(후속 증분):** 문서 템플릿(결정기록/런북/용어집), 관련 문서 추천, 그래프 뷰 시각화, 백엔드 link table(③에서 추가).

## 핵심 설계 결정

### D1. 클라이언트 파생 인덱스 (백엔드 변경 0)

백엔드 `GET /tree`는 이미 **권한 필터링된 전체 트리 + 본문**을 한 번에 내려준다 (`VaultService.tree(readable)` → `if (!readable.contains(row.id())) continue`; server 모드는 `guard.readableIds(user)` 전달). per-note GET 엔드포인트는 없으며 에디터가 메모리 트리의 `content`를 직접 읽는다.

따라서 백링크를 **클라이언트에서 계산**해도:
- 추가 다운로드 ≈ 0 (클라는 지금도 전체 읽기가능 트리+본문 보유 — 검색·에디터가 그 위에서 동작).
- 자동으로 권한-correct (못 읽는 노트는 애초에 트리에 없음).
- local·http 양쪽 모드에서 동일하게 동작.

백엔드 link table은 ③(에이전트 인제스션)에서 비로소 필요해질 때 추가한다.

### D2. id 기반 링크

링크는 노트 **이름이 아니라 안정 id**를 가리킨다 — CLAUDE.md 원칙 *"이름은 라벨일 뿐, rename은 권한 무영향"* 과 일관.

- **저장 형식:** `[[id:<nodeId>|<label>]]` — id가 진실, 파이프 뒤는 표시 라벨.
- **해석:** 저장 시점이 아니라 **렌더 시점에 현재 트리 대비 동적**으로 판정한다(resolvable / 연결 불가).
- **효과:** rename·이동해도 링크가 깨지지 않는다(Obsidian식 이름 기반의 고통 회피). ③ RAG export 때 id→title·path로 풀어 사람이 읽을 청크를 만든다.

### D3. 권한 모델 — 연결은 절대적, 가시성은 viewer 종속

> **edge(연결)는 절대적** — 작성자가 만든 사실. **가시성(traverse)은 viewer 종속** — 보는 사람마다 투영이 다르다.

work-note의 기존 deny-우선 권한 모델과 동형이다. 클라이언트 백링크는 *절대 그래프의 "내가 볼 수 있는 투영"* 이며, ③의 에이전트 RAG도 동일 원칙(server full 그래프를 질문자 권한으로 필터)으로 위아래가 일관된다.

**권한 회수 시 동작:**

| 대상 | 동작 |
|---|---|
| 링크 원문(`[[id:X\|label]]`) | **그대로 유지.** ACL 변경은 content를 재작성하지 않는다 |
| 권한 잃은 viewer의 렌더 | X가 그의 트리에 없음 → "연결할 수 없음"으로 비활성(클릭 X, X 본문 비노출) |
| 권한 복구 시 | **자동으로 다시 활성화**(클릭 가능) — 별도 복구/재작성 작업 없음 |
| 여전히 권한 있는 사용자 | 변화 없음, 링크 정상 동작 |

**금지:** 권한 회수 시 링크를 삭제·재작성하지 않는다. 권한은 viewer별·가변이므로 한 사람 기준으로 공유 문서 본문을 고치면 전원에게 손상되고, 권한이 복구돼도 되살아나지 않는다.

### D4. 접근불가 vs 삭제 — 의도적 비구분

클라이언트는 대상 id가 *"권한 없음"* 인지 *"삭제됨"* 인지 **구분할 수 없다**(둘 다 "내 트리에 없음"일 뿐). v1은 한 가지 중립 상태 **"연결할 수 없음"** 으로 표기한다. 둘을 구분하려면 존재 여부를 알려줘야 하는데, 그 자체가 *못 보는 노트의 존재 누출*이다. 정밀 구분이 필요하면 ③ 시대의 server resolve 엔드포인트 몫이다.

### D5. 라벨 노출 한계 (수용)

접근불가 대상은 렌더에서 **"🔒 연결할 수 없음"으로 표기하고 라벨을 숨긴다**(보수적 기본값). 단, 마크다운 *원문*에는 작성자가 입력한 라벨 텍스트가 남으며 source/export로는 보인다 — 이는 공유 문서에 일반 텍스트를 쓴 것과 동일한 노출이고, 새 권한 우회 채널이 아니다. 완전 차폐는 server-side 본문 sanitize라는 별개·과한 작업이라 비채택.

- `styles/app.css` — `.wikilink`(hover 진하게·`cursor: pointer`, 모노톤), `.wikilink.broken`(🔒·dimmed·`cursor: not-allowed`).

### 타입 (제안)

```ts
// lib/wikilink.ts
interface LinkRef {
  id: string;      // 대상 노트 id
  label: string;   // 표시 라벨(파이프 뒤, 없으면 빈 문자열)
  from: number;    // content 내 시작 오프셋
  to: number;      // content 내 끝 오프셋
}

// lib/linkIndex.ts
interface Backlink {
  sourceId: string;     // 이 노트를 가리키는 노트 id
  sourceTitle: string;  // 그 노트의 현재 title
}
type BacklinkIndex = Map<string /* targetId */, Backlink[]>;
```

### 링크 문법

- 표준 저장형: `[[id:<nodeId>|<label>]]`
- 라벨 생략형: `[[id:<nodeId>]]` → 렌더 시 대상 현재 title로 표시
- 파서는 `id:` 접두 + 알려진 id 패턴만 매칭. 매칭 실패(오타·수동 입력 깨짐) 시 원문 텍스트를 그대로 노출하고 파괴하지 않는다.

## 데이터 흐름

```
입력:   [[ → wikilinkComplete(flattenNotes 후보, title+path) → 선택 → content에 [[id:X|title]] 삽입
저장:   content PATCH (기존 1.5s 디바운스). 링크는 content의 일부라 별도 저장 없음
파생:   tree 변경 → buildBacklinks(tree) 재계산 (useMemo)
표시:   에디터       = wikilinkWidget(현재 tree로 resolvable 판정 → 토큰)
        미리보기·공유 = marked extension → a[data-note-id] 또는 broken span
        백링크 패널   = backlinks.get(activeId)
네비:   링크·백링크 클릭 → setActiveId(target) → 조상폴더 자동펼침 + scrollIntoView (이미 구현)
```

## 엣지 케이스 / 에러 처리

| 상황 | 처리 |
|---|---|
| 접근불가·삭제 대상(트리에 없음) | 중립 "연결할 수 없음" 비활성, 클릭 불가 (D4) |
| self-link | 허용하되 백링크 패널에서 self 제외 |
| 중복 title | autocomplete에 path 병기(`결제 / 런북 / 배포`)로 구분. 링크는 id라 모호성 없음 |
| 라벨 없는 `[[id:X]]` | 대상 현재 title로 표시(resolvable), 접근불가면 "🔒 연결할 수 없음" |
| 오타/깨진 직렬화 | 파서 매칭 실패 → 원문 텍스트 그대로 노출(파괴 안 함) |
| XSS | marked extension은 href 미생성·`data-note-id`만·라벨 escape (phase5 하드닝 일관) |
| 백링크 있는 노트 삭제 | "N개 문서가 이 노트를 참조 중" 정보성 경고(move-preview 모달 패턴). 휴지통이라 30일 복구 가능 |
