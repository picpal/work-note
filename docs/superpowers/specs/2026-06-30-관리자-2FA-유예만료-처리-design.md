# 관리자 2FA 유예 만료 처리 개선 — 설계

> 2026-06-30. 관련: [[2026-06-10 권한·디렉토리 설계]], 2FA 구현(merge be970db). 코드 검증 후 작성.

## 목표

관리자 2FA(TOTP) 등록 유예가 만료됐을 때, **AdminApp(admin.html)이 무한 로딩으로 멈추던 문제**를 해결하고, 만료된 관리자에게 명확한 강제 등록 화면 + 복구 안내(자가 등록 / 다른 관리자 초기화)를 제공한다. 더불어 관리자가 타 계정 2FA를 초기화할 때 **새 유예를 부여**해 즉시 재차단을 막는다.

## 배경 (현재 동작, 코드 검증)

- `AuthFilter`(backend): 유예 만료 + 2FA 미등록 관리자는 `ENFORCE_ALLOWLIST`(2FA 등록 경로 + `/api/auth/me` + logout) 외 **전부 403** `{"error":"2fa_enrollment_required"}`. 로그인은 의도적으로 허용 — 관리자가 **스스로 등록해 복구**하게 하기 위함(단독 관리자 영구 락아웃 방지).
- 메인 앱(`App.tsx:376~`): `mustEnrollNow(me.totp)`면 풀스크린 강제 등록 게이트(`SecurityTab`) 표시. `shouldNudge`면 유예 중 권고 배너. **정상 동작.**
- **AdminApp(`admin/AdminApp.tsx`)**: 이 게이트가 **없음**. `me()`는 allowlist라 200이지만 `reload()`(`/api/admin/users·roles·teams`)가 403 → `catch`가 토스트만 띄우고 `data`를 안 채움 → `loading = !me || !data`가 영구 true → **"불러오는 중" 무한**. ← 이번에 고칠 버그.
- (B) 관리자의 타 계정 2FA 초기화는 **이미 완비**: `AdminUserController POST /api/admin/users/{id}/2fa/reset`(`reset2fa` → `totpService.reset(id)` + 감사 `2fa.admin.reset`), 프런트 `AdminApi.resetTotp`, `Users.tsx` "2FA 초기화" 버튼(`totpEnabled`일 때)·확인 모달, 테스트(`api.test.ts`)까지. 단 `TotpService.reset`은 totp row 삭제 + 복구코드 무효화만 하고 **grace_start는 안 건드림** → 리셋 직후 대상이 즉시 `mustEnrollNow`(유예 만료 상태 유지).

## 범위

**포함:**
- (A) 강제 등록 게이트를 공유 컴포넌트로 추출해 메인 앱 + AdminApp 양쪽 적용.
- (A) 세션 중 유예 만료 방어 — 403 `2fa_enrollment_required` 전역 핸들러.
- (B) `reset2fa` 시 grace_start를 현재 시각으로 갱신(새 유예 부여).
- (B) 강제 게이트 안내문에 "다른 관리자에게 2FA 초기화 요청" 문구 추가.

**제외(YAGNI):**
- 미등록·유예만료 stuck 계정용 별도 "유예 연장" 관리자 액션 — 자가 등록이 1차 복구이므로 불필요.
- 로그인 화면 자체 차단 — 자가 등록 모델 유지(단독 관리자 영구 락아웃 방지).
- 신규 권한/스키마/런타임 의존성 — 없음.

## 컴포넌트 변경

### A1. `frontend/src/components/TotpEnrollGate.tsx` (신규)

App.tsx 인라인 게이트(현 376~403)를 표현 컴포넌트로 추출.

- Props: `{ totp: TotpInfo; onChanged: () => void; toast: (msg: string, icon?: string) => void; onLogout: () => void }`
- 렌더: `.totp-gate` 풀스크린 컨테이너 → `.totp-gate-box`(shield 아이콘 + "2단계 인증 등록 필요" 제목 + 안내문 + `SecurityTab` + 하단 로그아웃 버튼).
- 안내문(현 문구 유지 + 추가): "관리자 계정은 보안 정책에 따라 2FA(TOTP) 등록을 완료해야 계속 사용할 수 있습니다. 인증 앱을 준비하고 아래 절차를 따라 등록을 완료하세요. **인증 앱을 쓸 수 없으면 다른 관리자에게 2FA 초기화를 요청하세요.**"
- 스타일은 기존 인라인 style + `.totp-gate*` 클래스 그대로 이전. 새 CSS 없음.

### A2. `frontend/src/App.tsx`

- 인라인 게이트 블록(376~403)을 `createElement(TotpEnrollGate, { totp: me.totp, onChanged: () => AuthApi.me().then(setMe).catch(()=>{}), toast, onLogout: logout })`로 교체.
- 진입 조건 유지: `if (storageMode === "http" && me?.totp && mustEnrollNow(me.totp)) return <gate>`.
- `shouldNudge` 배너는 변경 없음.

### A3. `frontend/src/admin/AdminApp.tsx`

- import: `mustEnrollNow` (`../lib/totp2fa`), `TotpEnrollGate` (`../components/TotpEnrollGate`), `logout` 동작(기존 로그아웃 경로 재사용).
- 렌더 시작부, 기존 `loading` 게이트보다 **먼저**:
  ```
  if (me?.totp && mustEnrollNow(me.totp))
    return h(TotpEnrollGate, { totp: me.totp,
      onChanged: () => AuthApi.me().then(setMe).catch(()=>{}),
      toast: toastPush, onLogout: <기존 로그아웃> });
  ```
- `me`가 로드되면 `me.totp`로 게이트 판정 가능(=`reload()` 403을 기다릴 필요 없음). 이로써 hang 제거.
- `reload()`의 catch는 유지(다른 실패 대비)하되, 유예 만료 케이스는 게이트가 선점.

### A4. `frontend/src/api/http.ts`

- `on401`과 대칭으로 `on2faRequired` 추가:
  ```
  let on2faRequired: (() => void) | null = null;
  export function setOn2faRequired(h: (() => void) | null) { on2faRequired = h; }
  ```
- `req`/`reqForm`에서 `!res.ok`로 에러 본문 파싱 시: `status === 403 && body.error === "2fa_enrollment_required"`면 `on2faRequired?.()` 호출(그 후 기존대로 `ApiError` throw — 호출자 흐름 보존).
- 판정 순수 헬퍼 분리(테스트용): `export function is2faEnrollmentRequired(status: number, errorCode: string | undefined): boolean { return status === 403 && errorCode === "2fa_enrollment_required"; }`
- 등록처: App.tsx·AdminApp.tsx가 마운트 시 `setOn2faRequired(() => AuthApi.me().then(setMe))` → 세션 중 유예 만료 시 me 재조회로 게이트 노출. (login 앱은 미등록 — `on401`과 동일 원칙)

### B5. `backend` — reset2fa 시 grace 갱신

- `AdminUserController.reset2fa`: `totpService.reset(id)` 직후 대상의 `grace_start`를 현재 시각(주입된 `Clock` 기준 `LocalDateTime.now(clock)`)으로 갱신.
- 배치 위치: `UserAdminService`에 `resetGrace(id)` 추가(내부 `users.setGraceStart(id, now)`) 후 컨트롤러에서 `totpService.reset(id); svc.resetGrace(id);` 호출. (`TotpService`에 `UserMapper` 의존 추가 회피 — 응집도 유지)
  - `UserAdminService`는 현재 `Clock` 미주입 → 생성자에 `Clock` 추가하고 `LocalDateTime.now(clock)` 사용(`AuthController`·`AuthFilter`와 동일 패턴). `setGraceStart`는 ISO LocalDateTime 문자열 저장(기존 `totp_grace_start` 포맷과 동일).
- 감사 `2fa.admin.reset` 유지. 응답 204 유지.
- 효과: 리셋당한 사용자는 새 유예 기간 동안 정상 사용하며 재등록 가능(즉시 must-enroll 방지). 단 메인 앱/AdminApp 게이트는 유예 만료 시 여전히 강제(A로 정상 노출).

## 데이터 흐름 / 에러 처리

- 유예 만료 관리자 로그인 → `me()` 200(totp.graceExpired=true) → 게이트(메인 or admin) 표시 → `SecurityTab`에서 등록 → `onChanged`로 me 재조회 → 게이트 해제.
- 세션 중 만료 → 임의 API 403 `2fa_enrollment_required` → `on2faRequired` → me 재조회 → 게이트.
- me도 401(세션 사망) → 기존 `on401` → login.html.
- 다른 403(권한 거부)은 `is2faEnrollmentRequired`가 false → 영향 없음.

## 테스트 (프로젝트 관례: 순수함수 vitest, 실화면은 /qa)

- `is2faEnrollmentRequired(status, errorCode)` 순수 헬퍼 단위 테스트(403+코드 일치 true, 그 외 false).
- `mustEnrollNow`/`shouldNudge`는 기존 테스트 유지(`lib/totp2fa`).
- 백엔드: `reset2fa`가 `grace_start`를 갱신하는지 검증 — 기존 `AdminUser2faResetTest.java` 확장(리셋 전 만료 grace → 리셋 후 now로 갱신 + totp row 삭제 + 감사 기록).
- `TotpEnrollGate`는 표현 컴포넌트 → 유닛 테스트 없음. 실화면 회귀: admin.html 유예 만료 시 게이트 노출 + 등록 플로우 `/qa browse`.
