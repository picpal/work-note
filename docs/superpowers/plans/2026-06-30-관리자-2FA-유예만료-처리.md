# 관리자 2FA 유예 만료 처리 개선 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 관리자 2FA 유예 만료 시 AdminApp이 무한 로딩으로 멈추던 버그를 제거하고, 강제 등록 게이트를 메인 앱·AdminApp 공용 컴포넌트로 통일하며, 관리자의 타계정 2FA 초기화 시 새 유예를 부여한다.

**Architecture:** 기존 기계장치 재사용. (A) `App.tsx` 인라인 강제등록 게이트를 `TotpEnrollGate` 컴포넌트로 추출해 메인 앱·AdminApp 양쪽에 적용 + `http.ts`에 403 `2fa_enrollment_required` 전역 핸들러(세션 중 만료 방어, me 재조회로 loop-safe). (B) `reset2fa`가 `totpService.reset` 후 `grace_start`를 현재 시각으로 갱신.

**Tech Stack:** Vite 6 + TypeScript + React 18(JSX 미사용, `const h = React.createElement`) / Java 21 + Spring Boot 3.5 + MyBatis + SQLite / vitest, JUnit5+MockMvc.

## Global Constraints

- JSX 절대 미사용 — 모든 컴포넌트는 `React.createElement`(관례상 `const h = React.createElement`).
- 새 런타임 의존성 0. 새 권한·DB 스키마 0.
- 테스트 관례: 결정 로직은 순수 함수로 추출해 vitest 유닛 테스트. 표현(UI) 컴포넌트는 유닛 테스트 없이 `tsc`/빌드 + 실화면 `/qa` 회귀.
- 2FA 강제는 server 모드 + 관리자 한정(`mustEnrollNow(t) = t.enforced && t.graceExpired && !t.enabled`).
- `grace_start` 저장 포맷은 `LocalDateTime.now(clock).format(DateTimeFormatter.ISO_LOCAL_DATE_TIME)` (기존 `AuthController`와 동일).
- 기존 코드 패턴·파일 구조를 따른다. 불필요한 리팩터 금지.

```ts
/* 공유 fetch 코어 — VaultApi/AuthApi/AdminApi가 공용. 세션 쿠키는 same-origin 자동 전송. */
const BASE = "/api";

/** HTTP 상태코드를 보존하는 API 오류 — 호출부에서 status 기반 판별(예: 부트스트랩 409 허용)에 사용. */
export class ApiError extends Error {
  constructor(message: string, public status: number) {
    super(message);
    this.name = "ApiError";
  }
}

/** 401 전역 핸들러(세션 만료 → login.html). login 앱은 설치하지 않는다 — 로그인 실패 401이 리다이렉트가 되면 안 됨. */
let on401: (() => void) | null = null;
export function setOn401(handler: (() => void) | null) {
  on401 = handler;
}

/** 관리자 2FA 유예 만료(403) 전역 핸들러 — 세션 중 만료 시 호출부가 me를 재조회해 강제 등록 게이트를 띄운다. */
let on2faRequired: (() => void) | null = null;
export function setOn2faRequired(handler: (() => void) | null) {
  on2faRequired = handler;
}

/** 이 응답이 관리자 2FA 강제 등록 차단(403)인가 — AuthFilter가 보내는 코드와 일치할 때만 true. */
export function is2faEnrollmentRequired(status: number, errorCode: string | undefined): boolean {
  return status === 403 && errorCode === "2fa_enrollment_required";
}

/** !res.ok 공통 처리 — 본문 파싱 후 전역 핸들러 발화, ApiError throw. req/reqForm 공용(DRY). */
async function fail(res: Response): Promise<never> {
  const body = (await res.json().catch(() => ({}))) as { error?: string };
  if (res.status === 401 && on401) on401();
  if (is2faEnrollmentRequired(res.status, body.error) && on2faRequired) on2faRequired();
  throw new ApiError(body.error ?? `HTTP ${res.status}`, res.status);
}

export async function req<T>(
  path: string,
  init?: Omit<RequestInit, "headers"> & { headers?: Record<string, string> },
): Promise<T> {
  const res = await fetch(BASE + path, {
    ...init,
    headers: { "Content-Type": "application/json", ...init?.headers },
  });
  if (!res.ok) await fail(res);
  return res.status === 204 ? (undefined as T) : ((await res.json()) as T);
}

/** multipart 업로드 — Content-Type을 설정하지 않아 브라우저가 boundary를 채운다. (req와 동일 오류 처리) */
export async function reqForm<T>(path: string, form: FormData): Promise<T> {
  const res = await fetch(BASE + path, { method: "POST", body: form });
  if (!res.ok) await fail(res);
  return res.status === 204 ? (undefined as T) : ((await res.json()) as T);
}
```
